## 安装指南

### Windows

1. 下载 `ResumeToolkit-Setup.exe`
2. 双击运行安装程序
3. 按照向导完成安装
4. 桌面会出现「简历整理工具」快捷方式

### macOS

1. 下载 `ResumeToolkit.dmg`
2. 双击打开 DMG 文件
3. 将应用拖拽到「应用程序」文件夹
4. 从「启动台」或「应用程序」文件夹启动应用

### Linux

1. 下载 `ResumeToolkit.AppImage`
2. 给文件添加执行权限：
   ```bash
   chmod +x ResumeToolkit.AppImage
   ```
3. 双击运行，或通过命令行执行：
   ```bash
   ./ResumeToolkit.AppImage
   ```

## 快速开始

### 第一次使用

1. **启动应用**：双击桌面图标
2. **选择源目录**：点击「选择」按钮，选择包含简历的文件夹
3. **选择目标目录**：点击「选择」按钮，选择整理后简历的存放位置
4. **预览操作**：点击「预览」按钮，查看将要执行的操作
5. **确认执行**：预览结果无误后，点击「执行」按钮

### 示例场景

假设你的简历结构如下：

```
Downloads/
├── 简历备份/
│   ├── 张三_产品经理.pdf
│   ├── 李四_软件工程师.pdf
│   └── 重复_张三_产品经理.pdf
└── 新简历/
    └── 王五_数据分析师.pdf
```

**操作步骤**：
1. 源目录选择 `Downloads`
2. 目标目录选择 `Downloads/整理后的简历`
3. 点击「预览」
4. 看到结果：
   - 扫描到 3 个文件
   - 识别出 1 个重复文件
   - 将整理到对应分类文件夹
5. 点击「执行」

**最终结果**：
```
Downloads/整理后的简历/
├── 产品经理/
│   └── 张三_产品经理_本科_30_北京.pdf
├── 技术研发/
│   └── 李四_软件工程师_硕士_28_上海.pdf
└── 数据分析/
    └── 王五_数据分析师_本科_26_广州.pdf
```

## 详细操作

### 1. 设置路径

#### 源目录
- 包含所有简历的文件夹
- 支持任意深度嵌套
- 也可以选择包含多个简历文件夹的父目录

#### 目标目录
- 整理后的简历存放位置
- 会自动创建分类文件夹
- 如果目标目录不存在，会自动创建

**提示**：
- 首次使用建议新建一个空文件夹作为目标目录
- 可以随时修改源目录和目标目录
- 支持拖拽文件夹到输入框（未来版本）

### 2. 选择模式

#### 预览模式（推荐）
- 不实际移动/删除文件
- 模拟整个流程，显示将要执行的操作
- 可以反复预览，直到满意为止

#### 执行模式
- 真正执行整理操作
- 会移动文件、删除重复文件
- **不可撤销**（当前版本）

**安全建议**：
- 首次使用前，**备份**原始简历文件夹
- 先用预览模式查看结果
- 确认无误后再执行

### 3. 查看进度

处理过程中会显示实时进度：
- 当前步骤（扫描/解析/去重/归类）
- 处理百分比
- 当前处理的文件名

处理完成后显示统计：
- 扫描文件数
- 去重复数
- 移动数
- 重命名数
- 待人工审核数
- 错误数

### 4. 查看结果

整理完成后，打开目标目录，可以看到：
- 按职位分类的文件夹
- 标准化的文件名
- 待人工审核文件夹（解析不完整的文件）

## 高级配置

### 自定义分类

#### 添加新分类

1. 点击「分类管理」
2. 输入分类名称（如「人工智能」）
3. 输入关键词（如「AI, 机器学习, 深度学习」）
4. 点击「添加」

关键词用逗号分隔，文件名或职位名称包含任一关键词即可归类到此分类。

#### 查看现有分类

在「分类管理」中可以看到所有分类和对应的关键词。

#### 编辑/删除分类

当前版本暂不支持编辑和删除分类，可以通过编辑配置文件实现（见下节）。

### 编辑配置文件

配置文件位置：

| 平台 | 路径 |
|------|------|
| Windows | `%APPDATA%/resume-toolkit/config.json` |
| macOS | `~/Library/Application Support/resume-toolkit/config.json` |
| Linux | `~/.config/resume-toolkit/config.json` |

配置文件格式：

```json
{
  "customCategories": {
    "人工智能": ["AI", "机器学习", "深度学习", "NLP"],
    "区块链": ["区块链", "Blockchain"]
  },
  "customKeywords": {
    "技术研发": ["前端开发", "后端开发"]
  }
}
```

编辑配置文件后，重启应用生效。

## 常见问题

### Q1: 为什么有些文件没有分类？

**A**: 解析不完整的文件会放入「待人工审核」文件夹。可能的原因：
- 简历格式不规范
- 信息缺失（如没有姓名或职位）
- 扫描件 OCR 识别失败

**解决方法**：
- 手动编辑这些文件的信息
- 检查简历是否是标准格式
- 尝试转换格式（如 PDF 转 Word）

### Q2: 去重会误删文件吗？

**A**: 不会。去重基于文件内容哈希，只有完全相同的文件才会被认为是重复。

**安全建议**：
- 预览模式下会显示将要删除的重复文件
- 执行前仔细查看预览结果
- 首次使用前备份原始简历

### Q3: 能否撤销操作？

**A**: 当前版本不支持撤销。未来版本会添加此功能。

**替代方案**：
- 执行前备份原始文件夹
- 使用预览模式反复确认
- 发现问题后手动恢复

### Q4: 支持哪些简历格式？

**A**: 支持以下格式：
- PDF（包括可复制文字和扫描件）
- Word（.doc, .docx）
- 图片（.jpg, .jpeg, .png, .bmp）
- 网页（.html, .htm）

**注意**：扫描件需要 OCR 识别，可能不准确。

### Q5: 处理速度有多快？

**A**: 取决于以下因素：
- 文件数量
- 文件大小
- 文件格式（OCR 较慢）
- 电脑性能

**大致参考**：
- 纯文本文件：~100 个/秒
- 图片扫描件：~10 个/秒
- 处理 100 个文件：约 10-30 秒

### Q6: 能否批量处理多个文件夹？

**A**: 可以。选择包含多个简历文件夹的父目录作为源目录，工具会递归扫描所有子文件夹。

### Q7: 为什么需要访问文件系统权限？

**A**: 工具需要读取和移动文件，因此需要文件系统访问权限。这是桌面应用的标准行为。

**安全承诺**：
- 所有操作在本地完成
- 不上传任何数据到服务器
- 不收集用户信息

### Q8: 如何更新到最新版本？

**A**: 有两种方式：
1. 自动更新（未来版本）：应用会提示有新版本，点击即可更新
2. 手动更新：访问 [GitHub Releases](https://github.com/Liman-fully/resume-toolkit/releases) 下载最新版本

## 故障排除

### 问题 1：应用无法启动

**可能原因**：
- 操作系统版本过低
- 缺少系统依赖

**解决方法**：
- Windows: 需要Windows 10 或更高版本
- macOS: 需要macOS 10.14 或更高版本
- Linux: 需要主流发行版（Ubuntu 18.04+, Fedora 30+, 等）

### 问题 2：扫描不到文件

**可能原因**：
- 源目录路径错误
- 文件格式不支持
- 文件被系统锁定

**解决方法**：
- 确认源目录路径正确
- 检查文件扩展名是否在支持列表中
- 关闭可能占用文件的程序（如 PDF 阅读器）

### 问题 3：解析失败或信息不准确

**可能原因**：
- 简历格式不规范
- 扫描件质量差
- OCR 识别错误

**解决方法**：
- 尝试转换为可复制文字的格式
- 提高扫描件清晰度
- 手动编辑待人工审核的文件

### 问题 4：程序崩溃或卡死

**可能原因**：
- 内存不足
- 处理超大文件
- 系统资源紧张

**解决方法**：
- 关闭其他占用资源的程序
- 分批处理大量文件
- 重启应用或电脑

### 问题 5：移动文件失败

**可能原因**：
- 目标目录不可写
- 磁盘空间不足
- 文件名包含非法字符

**解决方法**：
- 检查目标目录权限
- 清理磁盘空间
- 重命名原文件（去除非法字符）
